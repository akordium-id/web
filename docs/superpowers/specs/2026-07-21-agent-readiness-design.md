# Agent-Readiness: Link Headers, Content Signals, Agent Skills Index

**Date:** 2026-07-21
**Bean:** `akdw-21lr`
**Site:** <https://akordium.id> (Astro static output, Cloudflare Pages)
**Status:** Approved (pending spec review)

## Context

`isitagentready.com` reported 11 "agent-readiness" goals as missing. The checklist is generic — it flags absence regardless of whether an item applies. Akordium's site is a **static marketing site with no API, no auth, and no MCP server**, so 6 of the 11 items are not applicable (they would advertise surfaces that do not exist):

| Item | Decision | Reason |
| ------ | ---------- | -------- |
| Link headers (RFC 8288) | ✅ Do | Static files + `_headers` |
| Content Signals (robots.txt) | ✅ Do | Trivial directive |
| Agent Skills index | ✅ Do | Publish one company-profile skill |
| Markdown for Agents | ⛔ Skip | Needs host-side content negotiation; out of chosen scope |
| API Catalog (RFC 9727) | ⛔ Skip | No API exists |
| OAuth/OIDC discovery | ⛔ Skip | No auth — stubs would mislead agents |
| OAuth Protected Resource | ⛔ Skip | No protected resources |
| Auth.md | ⛔ Skip | No auth |
| MCP Server Card | ⛔ Skip | No MCP server |
| WebMCP | ⛔ Skip | Not an interactive tool surface |
| DNS-AID | ⛔ Skip | DNS provider + DNSSEC, infra not code |

Publishing discovery metadata that points to non-existent OAuth/MCP endpoints is **actively harmful**: agents would attempt to authenticate against endpoints that 404. We deliberately keep scope to the items that describe real, existing resources.

## Design

### 1. Content Signals — `public/robots.txt` (edit)

Append one directive after the existing sitemap line:

```
Content-Signal: ai-train=no, search=yes, ai-input=yes
```

Per draft-romm-aipref-contentsignals. Rationale (marketing-friendly):

- `ai-train=no` — protect original case studies / blog from being used as model training data.
- `search=yes` — remain indexed by search engines (it is a marketing site).
- `ai-input=yes` — allow AI assistants to use the content at inference time to accurately describe Akordium's services (free, accurate reach).

This is a preference declaration, not enforcement.

### 2. Link response headers — `public/_headers` (new, Cloudflare Pages)

Cloudflare Pages reads a `_headers` file from the deploy root. Placing it under `public/` copies it into `dist/` on Astro build.

```
/*
  Link: </sitemap-index.xml>; rel="sitemap"; type="application/xml"
  Link: </.well-known/agent-skills/index.json>; rel="service-meta"; type="application/json"
```

- `rel="sitemap"` → existing sitemap (already generated by `@astrojs/sitemap`).
- `rel="service-meta"` (RFC 8631) → the agent-skills discovery index published in item 3.

Both target resources exist after this change, so the headers are accurate.

### 3. Agent Skills discovery index — `public/.well-known/agent-skills/` (new)

Per the Agent Skills Discovery RFC v0.2.0 (cloudflare/agent-skills-discovery-rfc).

**`index.json`** — discovery manifest:

```json
{
  "$schema": "https://agentskills.io/schema/0.2.0/agent-skills.json",
  "skills": [
    {
      "name": "akordium-profile",
      "type": "text/markdown",
      "description": "Company profile for Akordium Lab — services (PHP→Go and MySQL→PostgreSQL migration, MIS/dashboard builds) and how to engage/contact.",
      "url": "https://akordium.id/.well-known/agent-skills/akordium-profile.md",
      "sha256": "<digest of akordium-profile.md>"
    }
  ]
}
```

**`akordium-profile.md`** — the actual skill content. A concise, self-contained company profile so an agent fetching it understands what Akordium does and how a client engages. Content covers:

- Who Akordium Lab is (software studio, Surabaya, Indonesia).
- Core services: PHP→Go migration, MySQL→PostgreSQL (+TimescaleDB) migration, MIS/dashboard/internal-system builds.
- Notable work: MIS-APAR (CodeIgniter 3 → Go/PostgreSQL, ~50% server cost reduction), Orin GPS (10k+ device tracking on TimescaleDB), Bank Mega (automation/migration, 4x throughput).
- How to engage: email <hello@akordium.id>, WhatsApp +62 815-9988-995, contact form at <https://akordium.id/contact>.

The `sha256` digest is computed from the final bytes of `akordium-profile.md` and embedded into `index.json`.

## Verification

- `npm run build` succeeds and `dist/` contains `_headers`, `.well-known/agent-skills/index.json`, and the updated `robots.txt`.
- `sha256` in `index.json` matches `sha256sum dist/.well-known/agent-skills/akordium-profile.md`.
- After deploy (or via `wrangler pages dev`), `curl -I https://akordium.id/` shows the two `Link` headers.
- `curl https://akordium.id/robots.txt` shows the `Content-Signal` line.
- `curl https://akordium.id/.well-known/agent-skills/index.json` returns valid JSON with the schema + skills array.

## Out of Scope / Future

- **Markdown for Agents** — could be added later via a Cloudflare Worker or pre-rendered `.md` per route + `_headers` rewrite, if Faiq wants the blog/services served as markdown.
- **DNS-AID** — requires DNS provider access and DNSSEC; separate infra task outside this repo.
- If Akordium ever ships an API or MCP server, the corresponding discovery items (API catalog, OAuth metadata, MCP Server Card) become applicable and should be added then.
